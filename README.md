## ‚öôÔ∏è Technical Architecture

The project is built to be lightweight and easy to extend. We use **Phaser 3** as the engine.

### File Structure
*   **`index.html`**: The Game Engine. Handles:
    *   Rendering & Camera.
    *   Physics (Collisions).
    *   Input (WASD + Virtual Joystick for mobile).
    *   Level Loading Logic.
*   **`js/levels.js`**: The Data Layer. Contains:
    *   **Layouts:** ASCII-style arrays representing the map grids.
    *   **Portals:** Coordinates that trigger level changes (Doors/Stairs).
    *   **Metadata:** Colors, Names, Start positions.

### Current Features (v0.2)
- [x] **Movement System:** Smooth 8-directional movement with collision.
- [x] **Level Manager:** Dynamic loading of maps from data objects.
- [x] **Zone Transitions:** Fade-screen teleportation between zones.
- [x] **Mobile Support:** Touch-and-drag virtual joystick.
- [x] **CI/CD:** Auto-deployment to GitHub Pages via GitHub Actions.
